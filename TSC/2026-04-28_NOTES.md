## Immutable releases

Not for 1.12, but open the broader discussion for the next release. Scope should include core, forked providers, `tofu-ls`, and the VS Code extension. General stance: no republishing under the same tag — bump the version. Hardened by a recent go-releaser GPG-key bug where we declined to republish.

## Registry: commit-hash pinning and "discovered time"

- Registry now records the commit hash on first index of a module version and serves that hash as the ref — mirrors a Terraform registry change from a few years back, increases supply-chain security, prevents serving a re-published module under the same tag.
- Registry also now records a "discovered time" so we can later support "install latest, but lagged N days" in response to recent supply-chain incidents.
- Behavior needs documentation to preempt user confusion when a republished tag isn't served.

## Registry API documentation

Yousif: registry API is under-documented vs. the docs API; searches currently surface HashiCorp's registry docs first. Registry and `search.opentofu.org` API docs need to be unified and brought up to date with new fields, tied into sunsetting the `lib registry` project. Taken offline.
