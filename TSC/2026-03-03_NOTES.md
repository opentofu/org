## Provider Hashing RFC

- Christian presented the Provider Hashing RFC (https://github.com/opentofu/opentofu/pull/3434).
- RFC has morphed into three separate proposals: one large and two smaller tweaks.
- Good feedback received from other maintainers, Yousif, and Martin Atkins specifically.
- If implemented, 99% of the time users won't have to deal with `tofu providers lock` unless doing something very specialized.
- Terragrunt would be able to automatically write lock files, pulling both zip hashes and extracted/installed hashes from the registry in one go.
- Igor (Scalr) expressed strong support as this would allow removing the entire middleware layer from their runners related to caching and provider downloads. Requested backporting as far back as possible.
- Discussion on backporting:
  - Lock files will become larger with new H1 hashes when this is released, which could confuse users if it appeared in a patch version.
  - Christian is hesitant to backport due to the lock file changes being unexpected in a minor/patch release.
  - Roger suggested documenting criteria for when things get backported and what constitutes a breaking change.
  - Consensus: need to document a backporting policy before opening the floodgate.
- TSC members and TACOS providers encouraged to review the RFC.
